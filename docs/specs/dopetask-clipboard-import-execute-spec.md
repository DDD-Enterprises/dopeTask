# Spec: Clipboard Import and Execution Workflow

## Overview
The Clipboard Import workflow allows users to seamlessly ingest Task Packets from external planning sessions into the `dopeTask` DAG-aware series workflow.

## The Import Process
1. **Source:** A JSON Task Packet is copied to the system clipboard (usually generated by a Supervisor agent).
2. **Command:** `dopetask tp series import --auto`
3. **Validation:**
   - The CLI detects the OS and selects the appropriate clipboard backend (`pbpaste`, `xclip`, or `powershell`).
   - The content is parsed as strict JSON and validated against the `task_packet.schema.json`.
   - The command ensures the packet belongs to an active series (`packet.series` metadata).

## State Management
- **SERIES_STATE.json:** The authoritative source of truth for the series DAG.
- **Import Guard:** The system prevents importing packets that mismatch the series `base_branch` or have duplicate `id`s unless the `--overwrite` flag is used.
- **Location:** Imported packets are written to `out/task_packets/<TP_ID>.json`.

## Execution Trigger
Once imported, the `exec` command picks up the packet:
`dopetask tp series exec out/task_packets/<TP_ID>.json --agent gemini`

1. **Worktree Creation:** A temporary git worktree is created for the packet.
2. **Context Loading:** `SERIES_CONTEXT.json` is generated, providing the agent with proof of completion for parent dependencies.
3. **Implementation:** The `TaskExecutor` manages the implementation via the selected `Adapter`.
4. **Finalization:** Success triggers a commit in the worktree, which is then merged/promoted to the series branch.
